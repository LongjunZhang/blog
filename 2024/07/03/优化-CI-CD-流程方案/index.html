优化 CI/CD 流程方案

发表于 2024-07-03

背景描述

上篇文章介绍了 Docker 容器化部署了 CI/CD 组件的方案，文章的最后留了几个待优化的点——如何使 CI/CD 流程更加自动化。
本篇文章，将在上篇文章的基础上，介绍如何实现 CI/CD 流程的自动化。

实现目标

  1. 实现自动化流水线部署服务，开发者只需要关心代码的合并部分；一旦提交合并，流水线会完成服务的自动构建，自动部署，自动监控容器内服务启动状态，最终将结果通过企业微信或者钉钉的机器人告知开发者
  2. 模拟真实 Linux 服务器环境，实现容器化部署前端服务和后端服务

WebHook

为了实现 Jenkins Pipeline Stage 中启动 Ubuntu 服务器中的 docker-compose 构建脚本，以及监听 Ubuntu 服务器中容器内服务是否启动成功，需要在 Jenkins Pipeline Stage 中发起对远程服务器(Ubuntu) 的调用。
实现远程 RESTful 接口调用方式很多，这里我使用 SpringBoot 构建一个简单的 Web 服务去远程调用。

将该应用打包成 JAR 包，上传到 Ubuntu 服务器上，然后使用命令nohup ./jdk8/bin/java -jar webhook.jar > log 2>&1 &后台启动 SpringBoot 服务
注意事项：webhook.jar 必须运行在与startup-backend.sh, startup-frontend.sh脚本同一级目录下。
