Hexo+GitHub 搭建静态博客网站

发表于 2024-05-20

本文通过使用一个 GitHub 账号，实现搭建静态博客网站，实现网站自定义域名+网站托管+图床功能

使用技术栈

  1. Hexo：静态博客网站
  2. GitPage：提供网站托管，自定义域名
  3. PicGo：图床

Hexo

Hexo 是一个快速简介的博客框架, 详细介绍参见 Hexo 官网 . Hexo 官方文档有很详细的配置信息, 但这里不一一赘述, 只展现本教程所需要的内容.

安装环境

电脑上安装Git, Node.js软件, 注意 Node.js 版本, 这里建14.x.x版本

安装 Hexo

初始化 Hexo 网站

运行 Hexo 网站


访问 本地博客地址

自此, Hexo 个人博客网站的本地部署已完成, 接下来需要把博客部署到此公网上, 即可开始访问.

GitPage

GitPages 是 GitHub 提供的一项用于渲染静态网页的免费功能

创建仓库 & 开启 GitPages 功能

注意, 仓库要设置为Public状态, 否则后续网站无法访问



Save操作之后，等待片刻，刷新之后，便可以看到 GitPages 已经把我们网站部署上去了，点击公网访问

上传 Hexo 代码到仓库中

本地执行 Git 相关命令, 将代码推送到 GitHub 仓库


安装 hexo 提供的 GitPages 一键部署插件

修改_config.yml中的deploy配置

执行命令hexo clean && hexo deploy上传代码到 GitHub 仓库中

刷新 GitHub 仓库地址, 可以看到代码已经上传成功

稍等片刻, 等待 GitPages 渲染完成后, 刷新博客地址, 可以看到内容已经渲染出来, 但是网页样式布局没有渲染出来(这是由于路径不对, 导致 CSS, JS 相关文件没有获取成功, 后面解决该问题)

基本配置

使用 VSCode 打开工作空间hexo-test中的_config.yml修改博客基本信息, 更多配置参见 Hexo配置

基本操作

下面以新建一篇博客为例, 演示 Hexo 常见的操作

  1. 执行命令hexo new Hexo搭建教程.md, 将会在/sources/post目录下新建一篇文章
  2. 在 VSCode 中编写 Markdown 内容

  1. 部署到 GitPages 中
    等待 GitPages 渲染完毕, 刷新页面

PicGo

PicGo 是一款快速上传图片并获取图片 URL 的工具

安装配置 PicGo

  1. 在 PicGo 下载页面 选择合适的安装包, 本地安装
  2. 选中图床设置 => GitHub设置, 配置 GitHub 仓库的地址, 可以看到有 4 个必填项

  1. 创建一个新的 GitHub 仓库, 用于存储图片


点击 GitHub 右上角头像, 进入Settings => Developer Settings页面, 新增一个 Token, 用于 PicGo 客户端上传图片到 GitHub 的凭证





上传图片到 PicGo



此时, 粘贴面板上会自动复制一条 Markdown 格式的图片链接![]([https://raw.githubusercontent.com/wangqs999/blog-picgo/main/t-r-photography-TzjMd7i5WQI-unsplash.jpg)](https://raw.githubusercontent.com/wangqs999/blog-picgo/main/t-r-photography-TzjMd7i5WQI-unsplash.jpg))

Hexo 中使用

将 Markdown 格式的图片链接, 复制到文章中, 然后hexo clean && hexo g && hexo d部署到 GitHub 中, 等等渲染完毕, 即可访问
